# 2026-04-11 — Neuro layout

## Done

### Localisation — GUI rendering restored
- `loadFile(path)` helper added to [Neuro.Localisation.js](../apps/Localisation/Localisation.js): resolves the layout via `warp.layout(self.layout)`, pulls the `Blob` from `layout.apps[self.name].folder[path]`, `JSON.parse` for `.json` files, direct `eval` otherwise (runs in the app closure so loaded source sees `self`, `app`, `dom`, `warp`).
- `init()` deferred to a microtask (`Promise.resolve().then(init)`) so `appApi.layout` / `appApi.name`, set by App.js after the eval returns, are populated before `loadFile` reads them.
- `init` made private (no longer `this.init`) — only `reset`, `focusParameter`, `removeParameter`, `loadSet`, `selected`, `model` exposed.

### Localisation — drawer rendering & styling
- Missing `warp.gui.drawer` discovered at runtime. Added a local `makeDrawer(captionText, opts)` at the top of [gui.js](../apps/Localisation/gui.js) with `{ el, body, title, caption, open(), close(), isOpen() }` — scoped to the app, no engine touches.
- Fixed the `warp.dom.node` 3-arg bug throughout [gui.js](../apps/Localisation/gui.js): real signature is `(tag, attrs)`, so the previous `(tag, {}, { class })` calls were silently dropping every class — that's why nothing was styled and open/close had no visible effect.
- `drawerGui` outer `container` param was being shadowed by `let el` in the field loop, so drawers were appended to the last `<input>` instead of the container. Fixed to append to `container`.
- `scrollIntoView()` on drawer open was walking up all scrollable ancestors and pushing the 80px header div off-screen. Replaced with `container.scrollTop = drw.el.offsetTop - container.offsetTop` — only the `.neurovet` scroll container moves.
- Win caption reverted to plain text (`'NL Localisation'`) — [Win.js:152](../../../constructors/Win/Win.js#L152) uses `textContent`, HTML is escaped.
- Drawer head: `var(--md)` = 50px, `rgb(10,10,10)` background to match the header div, dark variant aligned.
- `.neurovet` made edge-to-edge (removed `margin-left: 5px`, `border-radius`, `padding-right`).
- Inputs/selects switched to `.sm` per engine sizing convention.

### Localisation — multi-select (`allowMultiple`)
- Ported from `examples_old/gui.js`: `select()` branches on `obj.allowMultiple` and renders a `<div class="multi-select-group">` with checkbox labels, exposes `getValues()` / `setValues()`, tags itself `_allowMultiple = true`.
- Render loop load-path uses `el.setValues(scope[prop])` when multi, `el.value` otherwise. Change wiring splits: per-checkbox `onchange` in the multi case, stores the value as an array in `self.selected`, toggles `.has-value` by count.
- Group title (parameter name) shown above the checkboxes, turns green when any value is selected. Label spacing bumped for breathing room.
- `.multi-select-group` CSS added to [Neuro.Localisation.css](../apps/Localisation/Localisation.css).

### Localisation — Tests support
- `this.loadSet(preset)` added: shallow-clones into `self.selected` and triggers private `init()` for a fresh re-render. Tests can call `(await warp.app('Neuro.Localisation')).loadSet(preset)`.

### Localisation — helpers & rules plumbing
- [helpers.js](../apps/Localisation/helpers.js) created *(Master)* — houses `matches`, `hasValue`, `displayValue`, pain helpers, `REFLEX_SEVERITY`, `worstReflex`, etc.
- Monolithic `rules.js` split into 17 files under [rules/](../apps/Localisation/rules/) *(Master)*.
- `init()` now loads `helpers.js` before any rule file so bare-name helpers land in the app closure, then loops a hardcoded ordered list of 17 rule basenames via `loadFile('rules/' + name + '.js')`, then mirrors `self.rules = app.rules`.
