Common rejection reasons — applies to both stores

PatternFix
"App seems incomplete" / "limited functionality"Reviewer can run a full case end-to-end on the demo account. Provide a 60-second walkthrough video in the review notes.
Privacy Policy URL broken or 404Live on a stable host. Test from an incognito browser before submitting. Footer-link it from the landing page too.
Marketing copy claims diagnosis or treatmentUse "decision support", "differential suggestion", "diagnostic aid". Never "diagnoses your pet" or "treats X".
Medical app without licensed-practitioner gateState clearly in the store description and on first launch that the tool is for licensed veterinarians. Acceptance toggle on first run.
Crash or login failure for reviewerSanity-test the demo account 24h before submission. If the backend is down for the reviewer, it is auto-rejection.

Backend requirements that the apps depend on

These apply regardless of which platform the user installs from — see publish-web.html for the actual hosting / database choices.

  • HTTPS on every endpoint (TLS managed by Vercel / Cloudflare).
  • Authentication system that works the same way on web and native (email + password, magic link).
  • Subscription state synced from Stripe (web) and IAP receipts (iOS / Android) — single source of truth in our database.
  • Daily backups of the subscription / user table from day one.
  • Public Privacy Policy + Terms URLs reachable from any platform.

Order of operations across platforms

  1. Brand decided → register domains → reserve store names — see publish-web.html for domain detail.
  2. Apple Developer + Google Play accounts created in parallel — DUNS lead time for Apple (1–2 weeks) starts immediately.
  3. Privacy Policy + Terms live (see legal.html) → necessary for any submission.
  4. PWA + Stripe live on a real device end-to-end — this is the launch vehicle, not just a fallback.
  5. Android submission — typically approved in days.
  6. iOS submission — expect 1–2 review rounds. Soft launch on web + Android does not wait for Apple.
