Scaffolding

Warp CSS is a small, fast prototyping framework. No build step, no preprocessor. Just classes on elements.
Grid
.r2 – .r10 create equal-column grids. .c2 – .c10 span columns. Nest freely.
<div class="r3">
  <div class="s1 p5">A</div>
  <div class="c2 s2 p5">B (spans 2)</div>
</div>
Sizes
Five sizes: xs sm md lg xl. Put on a container — all buttons, inputs, selects inside inherit it. md is default.
<div class="r2 xs">
  <input placeholder="small">
  <button class="s1">Go</button>
</div>
Padding & margin
Fixed steps: 0, 5, 10, 15, 20 px. Directional: pt pb pl pr, mt mb ml mr. Negatives: .m-5 .mt-10. Auto: .mla .mra.
.p0 .p5 .p10 .p15 .p20
.pt0 .pb5 .pl10 .pr20

.m0 .m5 .m10 .m15 .m20
.mt-5 .m-10 .mla .mra
Colours
Background: .s0–.s6 (default, primary, secondary, success, warning, danger, link). Text only: .t0–.t6. Driven by skin CSS variables.
<button class="s1 sm">Primary</button>
<span class="t5">Error text</span>
<div class="s3 p10">Success</div>
Height & width tokens
.hxs–.hxl and .wxs–.wxl map to --xs(25px) through --xl(100px). Use for fixed-size elements.
<div class="s1 hmd wmd">Square</div>
<button class="s5 hxs wxs round">−</button>
Other helpers
.round circle, .raised .raised-sm .inset-xs elevation, .op0–.op9 opacity, .disabled .hidden state, .clear strips bg/border.
<button class="xs round s1">+</button>
<div class="s0 p10 raised">Card</div>
<span class="op5">50% opacity</span>

gui.toast( text, time? )
Shows a dark notification at the bottom-center. Auto-dismisses after time ms (default 3000). Pass 0 for permanent. Duplicates bump a count badge.
warp.gui.toast("Hello!");

// custom duration (5s)
warp.gui.toast("Slow toast", 5000);

// permanent — user must dismiss
warp.gui.toast("Sticky", 0);
gui.error( text )
Shows a danger-coloured notification at the bottom-right. Same stacking and dedup as toast. Auto-dismisses after 3 s.
warp.gui.error("Something went wrong");

// uses --dangerBg / --dangerColor
// anchored bottom-right, stacks upward
gui.alert( text )
Centered modal with backdrop. Has an OK button. Dismissed by clicking OK or the backdrop.
warp.gui.alert("Heads up!");

// centered, dark background
// closes on OK or backdrop click
// destroyOnHide: true
gui.confirm( text ) → Promise<boolean>
Async centered modal. Resolves true on OK, false on Cancel or backdrop.
var ok = await warp.gui.confirm("Delete?");
if (ok) {
    // proceed
}
gui.input( valueOrText, text? ) → Promise<string|false>
Async centered modal with a text input. One arg = message (empty input). Two args = default value + message. Resolves with new value if changed, false if unchanged or cancelled.
// empty input with message
var name = await warp.gui.input("Enter name:");

// pre-filled input
var val = await warp.gui.input("old value", "Rename:");
// val = new string or false
layout.destroy() + gui.confirm
Use gui.confirm before destroying a layout to let the user review what will be removed — layout name plus its apps.
var layout = await warp.layout("Test");
var apps = Object.keys(await layout.app());
var ok = await warp.gui.confirm(
    "Destroy Test and " + apps.length + " apps?"
);
if (ok) await layout.destroy();
warp.load() — reload existing layout
If a layout is already loaded, confirm before destroying and reloading it. warp.load returns the existing instance if the name matches — destroy first to force a fresh load.
var existing = warp.lists.instances.layouts["Test"];
if (existing) {
    var ok = await warp.gui.confirm("Reload Test?");
    if (ok) await existing.destroy();
    else return;
}
var layout = await warp.load("../dist/Test.layout");
